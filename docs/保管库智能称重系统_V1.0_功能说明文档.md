# 保管库智能称重系统 V1.0 功能说明文档

**文档版本：** V1.0  
**发布日期：** 2026年3月13日  
**编制单位：** 系统开发团队  

---

## 1. 系统概述

保管库智能称重系统是一套面向金融机构保管库场景的实时称重监控与管理平台。系统通过RS485网关和称重仪表对保管库内保险柜组进行7×24小时不间断的重量监测，实时采集各柜组的重量数据，当检测到异常重量变化时自动触发报警，从而保障保管库内贵重物品的安全。

系统采用B/S架构，前端基于React 19 + Tailwind CSS 4构建现代化Web界面，后端基于Node.js + Express + tRPC提供类型安全的API服务，数据库采用MySQL 8.0进行数据持久化存储。系统支持局域网内多终端同时访问，无需安装客户端软件。

### 1.1 系统架构

系统由三个核心服务组成，协同工作完成从数据采集到前端展示的完整链路。

| 服务名称 | 功能定位 | 技术栈 | 通信方式 |
|---------|---------|--------|---------|
| Web主服务 | 前端页面渲染、API接口、WebSocket中转 | React 19 + Express + tRPC | HTTP / WebSocket |
| 采集服务 | 硬件通信、数据采集、报警检测 | Node.js + Modbus-RTU | TCP / WebSocket |
| MySQL数据库 | 数据持久化存储 | MySQL 8.0 | TCP 3306 |

数据流向为：称重仪表 → RS485网关（TCP透传） → 采集服务（Modbus-RTU解析） → WebSocket推送至Web主服务 → 持久化到数据库 + 实时推送至前端浏览器。前端通过WebSocket直连接收实时数据，延迟在500毫秒以内。

### 1.2 硬件架构

系统支持的硬件设备层级结构如下：

| 设备层级 | 设备类型 | 说明 |
|---------|---------|------|
| 第一层 | RS485网关（如ZLAN6808） | 串口服务器，负责RS485到TCP/IP的协议转换，每个网关可配置多个COM端口 |
| 第二层 | COM端口 | 网关上的物理串口，每个COM端口独立配置IP地址、网络端口、波特率等串口参数 |
| 第三层 | 称重仪表（DY7001/DY7004） | 连接在COM端口上的Modbus从站设备，DY7001为单通道、DY7004为四通道 |
| 第四层 | 仪表通道（CH1~CH4） | 仪表的物理测量通道，每个通道连接一个称重传感器 |

## 2. 功能模块详细说明

### 2.1 用户登录与认证

系统采用本地账户体系，支持用户名和密码登录。用户登录后系统签发JWT令牌，通过Cookie维持会话状态。系统预置一个默认管理员账户（用户名：admin，初始密码：admin123），首次使用后建议立即修改密码。

系统支持两种用户角色：**管理员（admin）** 拥有系统全部权限，可管理用户、分配权限、配置设备；**操作员（operator）** 的权限由管理员按模块分配，可精确控制每个功能模块的查看权限和操作权限。

### 2.2 实时监视

实时监视是系统的核心功能页面，提供保管库内所有柜组重量状态的全局可视化视图。

**2D布局可视化：** 系统通过导入保管库的DXF图纸文件，将柜列位置信息提取并渲染为2D SVG平面图。每个柜列可绑定到对应的保险柜组，绑定后的柜组以不同颜色标识，系统从20种预设颜色中自动循环分配，确保视觉区分度。布局支持整图旋转（0~360度），以适应不同的屏幕显示方向。

**实时重量显示：** 系统通过WebSocket直连接收采集服务推送的实时数据。当鼠标悬浮在某个柜组上方时，弹出信息面板显示柜组的详细信息，包括柜组名称、所属区域、初始重量、当前实时重量、重量变化值以及在线状态。当前重量根据绑定通道的实时采集数据持续刷新，刷新延迟约500毫秒。

**状态颜色标识：** 柜组根据当前状态显示不同颜色。正常状态下使用柜组的专属分配色，警告状态显示为黄色，报警状态显示为红色并伴有闪烁效果，离线状态显示为灰色。

**报警面板：** 页面右侧提供滑出式报警面板，实时显示未处理的报警列表（最新在上），支持报警确认和忽略操作。报警面板顶部显示未处理报警总数，新报警产生时可触发声音提示（可配置开关）。

### 2.3 网关配置

网关配置模块用于管理RS485串口服务器及其COM端口。

**网关管理：** 支持网关的增删改查操作，可配置网关名称、型号和备注信息。网关列表显示每个网关的在线状态，该状态由其所有COM端口的连接状态综合计算——所有COM端口均在线时网关显示在线，任一COM端口离线则网关显示离线。

**COM端口配置：** 在网关的添加/编辑对话框中内嵌COM端口配置列表。每个COM端口独立配置以下参数：

| 参数名称 | 说明 | 默认值 |
|---------|------|--------|
| IP地址 | COM端口对应的网络IP | - |
| 网络端口 | TCP端口号 | - |
| 波特率 | 串口通信速率 | 19200 |
| 数据位 | 数据位长度 | 8 |
| 停止位 | 停止位数量 | 1 |
| 校验位 | 奇偶校验方式 | None |
| 协议类型 | 通信协议 | Modbus RTU |
| 通信超时 | 超时时间（毫秒） | 1000 |
| 重试次数 | 失败重试次数 | 3 |
| 采集频率 | 数据采集间隔（毫秒） | 500 |

**采集服务控制：** 网关配置页面顶部显示采集服务的连接状态指示器，并提供"重载采集配置"按钮。当网关或COM端口配置发生变更后，可通过此按钮通知采集服务热重载配置，无需重启服务。

### 2.4 仪表配置

仪表配置模块用于管理称重仪表及其测量通道。

**仪表管理：** 支持仪表的增删改查操作。创建仪表时需指定设备编码（如C001）、仪表型号（DY7001或DY7004）、从站地址（Modbus SlaveId，1~247）以及所属COM端口。COM端口通过右侧树形选择面板选择，以网关为父节点、COM端口为子节点的层级结构展示。系统自动检测同一COM端口下的从站地址冲突。

**通道管理：** 仪表创建后系统根据型号自动生成对应数量的通道（DY7001生成CH1，DY7004生成CH1~CH4），通道数量固定不可增减。仪表列表采用可展开的树形结构，点击仪表行的展开箭头可查看其下属通道。每个通道支持以下配置：

| 配置项 | 说明 |
|--------|------|
| 通道标签 | 显示名称，默认CH1~CH4 |
| 启用状态 | 启用/禁用该通道的数据采集 |
| 校准系数（scale） | 实际值 = rawValue × scale + offset |
| 校准偏移（offset） | 校准公式中的偏移量 |
| 单位 | 测量单位，默认kg |
| 精度 | 小数位数，默认2位 |

**获取当前值：** 每个通道提供"获取当前值"按钮，点击后通过TCP直连网关实时读取仪表通道的当前测量值，读取成功后在"当前值"列中显示结果。

### 2.5 保险柜组管理

保险柜组管理模块用于管理保管库内的保险柜组及其与仪表通道的绑定关系。

**柜组管理：** 支持柜组的增删改查操作，可配置柜组名称、所属区域、初始重量（kg）和报警阈值（kg）。柜组列表采用可展开的树形结构：第一层为柜组信息，展开后第二层显示绑定的仪表信息，再展开第三层显示仪表的通道信息，各层级有明确的缩进区分。

**通道绑定：** 柜组通过绑定配置功能关联仪表通道。绑定界面采用右侧树形面板，以仪表为父节点、通道为子节点的层级结构展示，支持一次选择多个通道批量完成绑定。每个绑定关系可配置权重系数（coefficient）和偏移量（offset），柜组的实时重量按以下公式计算：

> **柜组重量 = Σ(通道值_i × 系数_i) + Σ(偏移量_i)**

系统自动进行绑定校验，包括通道启用状态检查、仪表在线提示以及重复绑定检测（同一通道默认不允许被多个柜组引用）。

### 2.6 数据记录

数据记录模块展示所有柜组的重量变化历史。系统在实时采集过程中，当检测到柜组重量发生变化（变化量超过0.001kg且距上次记录超过5秒）时，自动写入重量变化记录。

每条记录包含以下信息：序号、记录日期、记录时间、变化前重量、变化后重量、变化值以及是否触发报警。支持按柜组筛选和关键词搜索，列表按时间倒序排列，支持分页浏览。

### 2.7 报警管理

报警管理模块提供报警记录的查询、处理和统计功能。

**报警触发机制：** 当柜组的瞬间重量变化超过设定的报警阈值时，系统自动生成超重报警记录。采集服务内置报警去重逻辑，相同柜组的相同类型报警在一定时间内只记录一次（增加发生次数计数器），避免重复报警。

**报警处理：** 报警记录支持三种处理状态——未处理（pending）、已确认（handled）和自动解除（auto_resolved）。操作人员可对未处理的报警执行确认或忽略操作，操作记录写入报警处理日志。当柜组重量恢复正常时，系统自动将报警标记为自动解除状态，但仍需工作人员确认。

**报警统计：** 页面顶部显示报警统计摘要，包括未处理报警数量和今日报警总数。

### 2.8 数据分析

数据分析模块提供多维度的数据可视化图表，帮助管理人员了解系统运行状况。

| 图表类型 | 展示内容 |
|---------|---------|
| 历史重量趋势图 | 各柜组在指定时间范围内的重量变化趋势 |
| 变化统计图 | 重量变化事件的频次分布和统计分析 |
| 设备状态统计 | 各设备的在线/离线状态占比 |
| 报警统计分析 | 报警事件的类型分布、时间分布和处理情况 |

### 2.9 审计日志

审计日志模块记录系统中所有关键操作的完整审计轨迹。每条日志包含操作人、操作时间、操作类型、操作对象类型、操作对象ID以及变更摘要。支持的操作类型包括创建、修改、删除和绑定变更等。日志列表支持分页浏览和时间范围筛选。

### 2.10 用户管理

用户管理模块供管理员管理系统用户和权限分配。

**用户管理：** 管理员可新建操作员账户（设置用户名、密码和角色）、编辑操作员信息（修改用户名、重置密码）以及删除操作员账户。用户列表按角色分组显示，管理员组和操作员组分别列出。

**权限配置：** 管理员可为每个操作员按模块配置权限。系统包含以下10个功能模块，每个模块可独立配置查看权限和操作权限：

| 模块标识 | 模块名称 | 说明 |
|---------|---------|------|
| dashboard | 实时监视 | 保管库实时监控视图 |
| gateway_config | 网关配置 | RS485网关和COM端口管理 |
| instrument_config | 仪表配置 | 称重仪表和通道管理 |
| cabinet_group | 保险柜组 | 柜组管理和通道绑定 |
| data_records | 数据记录 | 重量变化历史记录 |
| alarm_management | 报警管理 | 报警查询和处理 |
| data_analysis | 数据分析 | 数据可视化图表 |
| audit_logs | 审计日志 | 操作审计轨迹 |
| user_management | 用户管理 | 用户和权限管理 |
| layout_editor | 布局编辑器 | 保管库布局设计 |

操作员登录后，侧边栏仅显示已授权的模块菜单项，未授权的页面在路由层面进行拦截。

**密码管理：** 所有用户均可通过侧边栏底部用户菜单中的"修改密码"选项修改自己的登录密码。

### 2.11 布局编辑器

布局编辑器模块用于设计和管理保管库的2D平面布局。

**DXF导入：** 支持导入AutoCAD DXF格式的保管库平面图文件。系统自动解析DXF文件中带叉号标记的矩形区域，将其识别为柜列位置并渲染到2D SVG画布上。

**柜组绑定：** 在画布上通过鼠标框选一个或多个柜列，然后从柜组列表中选择目标柜组进行绑定。绑定后的柜列自动着色，柜组名称以带色标签形式叠加在选中区域的中心位置。系统从20种预设颜色中自动循环分配，确保不同柜组之间的视觉区分度。

**布局管理：** 支持多个布局方案的保存和加载。每个布局方案记录名称、描述、柜列数据和绑定关系。系统通过"激活"机制确定实时监视页面使用哪个布局方案，同一时间只能有一个激活的布局。

**整图旋转：** 支持0~360度的整图旋转功能，旋转角度通过带上下按钮的数字输入框控制（颗粒度为1度），旋转角度保存在布局数据中。

### 2.12 系统设置

系统设置模块提供数据库记录自动清理功能的配置界面。

**数据统计：** 页面顶部展示各数据表的当前记录数量，帮助管理员了解数据库的存储状况。

**清理策略配置：** 系统对5张持续增长的数据表提供双重清理保护策略——按时间过期和按最大记录数上限。各表的默认配置如下：

| 数据表 | 默认保留天数 | 默认最大记录数 | 增长速度估算 |
|--------|------------|--------------|------------|
| 采集数据（collectionData） | 30天 | 500万条 | 约69万条/天（4通道×500ms） |
| 重量变化记录（weightChangeRecords） | 365天 | 200万条 | 约1.7万条/天 |
| 报警记录（alarmRecords） | 365天 | 不限 | 视报警频率而定 |
| 报警处理日志（alarmLogs） | 365天 | 不限 | 视处理频率而定 |
| 审计日志（auditLogs） | 365天 | 不限 | 视操作频率而定 |

**自动清理任务：** 系统后台运行定时清理任务，每小时检查一次，在配置的执行时间自动执行清理。管理员也可在页面上手动触发立即清理。清理功能默认未启用，需管理员在系统设置页面手动开启。

## 3. 技术规格

### 3.1 系统环境要求

| 项目 | 要求 |
|------|------|
| 服务器操作系统 | Ubuntu 22.04 LTS 或更高版本 |
| Node.js | v22.x 或更高版本 |
| MySQL | 8.0 或更高版本 |
| 内存 | 建议4GB以上 |
| 磁盘空间 | 建议50GB以上（含数据库存储） |
| 网络 | 局域网环境，需与RS485网关网络互通 |

### 3.2 浏览器兼容性

| 浏览器 | 最低版本 |
|--------|---------|
| Google Chrome | 90+ |
| Microsoft Edge | 90+ |
| Mozilla Firefox | 90+ |
| Safari | 15+ |

### 3.3 通信协议

| 协议 | 用途 | 端口 |
|------|------|------|
| HTTP | Web页面和API访问 | 3000（默认） |
| WebSocket | 实时数据推送 | 与HTTP共用端口 |
| Modbus RTU over TCP | 仪表数据采集 | 由COM端口配置决定 |
| MySQL | 数据库连接 | 3306（默认） |

### 3.4 数据库表结构概览

系统共包含14张数据库表，按功能分为以下几组：

| 分组 | 表名 | 用途 |
|------|------|------|
| 用户与权限 | users | 用户账户信息 |
| | userPermissions | 用户模块权限配置 |
| 硬件配置 | gateways | RS485网关信息 |
| | gatewayComPorts | 网关COM端口配置 |
| | weighingInstruments | 称重仪表信息 |
| | instrumentChannels | 仪表测量通道 |
| 业务数据 | cabinetGroups | 保险柜组信息 |
| | groupChannelBindings | 柜组与通道绑定关系 |
| | vaultLayouts | 保管库布局方案 |
| 运行数据 | collectionData | 采集数据历史 |
| | weightChangeRecords | 重量变化记录 |
| | alarmRecords | 报警记录 |
| | alarmLogs | 报警处理日志 |
| 系统管理 | auditLogs | 操作审计日志 |
| | systemSettings | 系统配置参数 |
| | deviceConnectionStatus | 设备连接状态 |
