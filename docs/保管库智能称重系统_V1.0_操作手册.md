## 1. 系统概述

### 1.1 系统简介

保管库智能称重系统是一套面向金融保管库的7×24小时实时称重监控与智能报警管理平台。系统通过称重传感器和RS485网关采集保管库内保险柜组的实时重量数据，经过服务端处理后在Web界面上以2D可视化布局实时展示，并在重量异常时自动触发报警。

### 1.2 角色与权限

系统采用双角色体系，不同角色拥有不同的操作权限：

| 功能模块 | 管理员（Admin） | 操作员（Operator） |
|---------|:-------------:|:-----------------:|
| 实时监视 | 完全访问 | 完全访问 |
| 硬件配置（网关/仪表） | 完全访问 | 无权限 |
| 柜组管理 | 完全访问 | 无权限 |
| 布局编辑 | 完全访问 | 无权限 |
| 报警管理 | 完全访问 | 确认/忽略 |
| 数据记录 | 完全访问 | 只读查看 |
| 数据分析 | 完全访问 | 只读查看 |
| 审计日志 | 完全访问 | 无权限 |
| 用户管理 | 完全访问 | 无权限 |
| 系统设置 | 完全访问 | 无权限 |

### 1.3 系统访问方式

在局域网内通过浏览器访问系统。推荐使用 Chrome、Edge 或 Firefox 的最新版本。在浏览器地址栏输入服务器的IP地址和端口号即可访问，例如 `http://192.168.1.100:3000`。

### 2.1 登录系统

打开浏览器，输入系统地址后将看到登录页面。点击"登录"按钮，系统将跳转至统一认证页面完成身份验证。认证成功后自动返回系统主界面。

系统采用左侧边栏导航 + 右侧内容区域的经典布局：

**左侧边栏包含以下菜单项（管理员视角）：**

## 3. 实时监视操作

### 3.1 进入实时监视页面

点击左侧边栏的"实时监视"菜单项，进入保管库实时监视页面。该页面是系统的核心功能页面，提供保管库全局态势的实时可视化展示。

### 3.2 界面元素说明

实时监视页面包含以下主要元素：

**顶部信息栏：** 显示保管库名称、当前时间、连接状态指示器（绿色Wifi图标表示WebSocket实时连接正常）。

**中央区域 — 2D平面图：** 以2D SVG平面图的形式展示保管库的物理布局。每个柜组以色块表示，色块颜色反映柜组当前状态：

| 状态 | 颜色表示 | 含义 |
|------|---------|------|
| 正常 | 柜组专属颜色 | 重量在正常范围内 |
| 警告 | 黄色 | 重量接近报警阈值 |
| 报警 | 红色（闪烁） | 重量变化超过报警阈值 |
| 离线 | 灰色 | 绑定的仪表通道离线 |

**右侧报警面板：** 实时显示未处理的报警信息，包括报警柜组名称、报警时间和重量变化值。

### 3.3 查看柜组详情

将鼠标悬浮在任意柜组色块上方，系统将弹出悬浮详情面板，显示以下信息：

- **柜组名称** — 柜组的标识名称
- **所属区域** — 柜组所在的区域位置
- **初始重量** — 柜组的基准初始重量（kg）
- **当前重量** — 实时采集计算的当前重量（kg），约500毫秒刷新一次
- **重量变化** — 当前重量与初始重量的差值
- **在线状态** — 绑定通道的在线/离线状态

### 3.4 处理报警

当右侧报警面板显示未处理的报警时，点击报警条目可查看报警详情。操作人员可执行以下操作：

- **确认报警** — 表示已知悉该报警并采取了相应措施，需填写处理备注
- **忽略报警** — 表示该报警为误报或已自行恢复，需填写忽略原因

### 3.5 声音提示

系统支持报警声音提示功能。当新报警产生时，系统会播放提示音。可通过页面上的声音开关按钮控制是否启用声音提示。

## 4. 硬件配置管理

> **注意：** 硬件配置管理仅限管理员操作。

### 4.1 网关管理

#### 4.1.1 查看网关列表

点击"硬件配置 → 网关管理"进入网关管理页面。页面以表格形式展示所有已配置的RS485网关，包括网关名称、型号、在线状态等信息。

#### 4.1.2 添加网关

1. 点击页面右上角的"添加网关"按钮
2. 在弹出的对话框中填写以下信息：
   - **网关名称** — 为网关指定一个易于识别的名称
   - **网关型号** — 选择网关型号（如 ZLAN6808）
   - **备注** — 可选，填写网关的安装位置或其他说明
3. 点击"确定"保存

#### 4.1.3 配置COM端口

每个网关可包含多个COM端口。在网关列表中，点击某个网关的展开按钮可查看其下属的COM端口列表。

**添加COM端口：**

1. 点击网关行右侧的"添加端口"按钮
2. 填写以下参数：

| 参数 | 说明 | 示例值 |
|------|------|--------|
| 端口名称 | COM端口标识名称 | COM1 |
| IP地址 | 网关的网络IP地址 | 192.168.1.200 |
| 网络端口 | TCP通信端口号 | 4001 |
| 波特率 | 串口通信速率 | 9600 |
| 数据位 | 数据位长度 | 8 |
| 停止位 | 停止位长度 | 1 |
| 校验位 | 奇偶校验方式 | None |
| 协议类型 | 通信协议 | Modbus RTU |
| 超时时间 | 通信超时（毫秒） | 1000 |
| 重试次数 | 通信失败重试次数 | 3 |
| 采集频率 | 数据采集间隔（毫秒） | 500 |

3. 点击"确定"保存。配置变更后采集服务将自动检测并重新建立连接，无需手动重启。

#### 4.1.4 编辑与删除

- **编辑：** 点击网关或COM端口行右侧的编辑图标，修改后保存
- **删除：** 勾选要删除的条目，点击"删除"按钮。删除网关将同时删除其下属的所有COM端口

### 4.2 仪表管理

#### 4.2.1 查看仪表列表

点击"硬件配置 → 仪表管理"进入仪表管理页面。页面以表格形式展示所有已配置的称重仪表及其通道信息。

#### 4.2.2 添加仪表

1. 点击"添加仪表"按钮
2. 填写以下信息：
   - **仪表名称** — 为仪表指定标识名称
   - **仪表型号** — 选择型号：DY7001（单通道）或 DY7004（四通道）
   - **从站地址** — Modbus从站地址（1~247），系统会自动检测同一COM端口下是否存在地址冲突
   - **所属COM端口** — 通过树形面板选择该仪表连接的COM端口（先选网关，再选端口）
   - **备注** — 可选说明信息
3. 点击"确定"保存

保存后，系统将根据仪表型号自动生成对应数量的通道（DY7001生成CH1，DY7004生成CH1~CH4）。通道数量固定，不可手动增减。

#### 4.2.3 配置通道

在仪表列表中展开某个仪表，可查看其下属通道。每个通道可配置以下参数：

- **通道标签** — 自动生成（CH1、CH2等），可自定义修改
- **启用状态** — 开启或关闭该通道的数据采集
- **校准系数** — 通道值的乘法校准因子（默认1.0）
- **偏移量** — 通道值的加法偏移（默认0）
- **单位** — 测量单位（默认kg）
- **精度** — 小数位数（默认2位）

直接在通道行中修改参数后，点击保存图标即可生效。

## 5. 柜组管理操作

> **注意：** 柜组管理仅限管理员操作。

### 5.1 查看柜组列表

点击"柜组管理"菜单项进入柜组管理页面。页面以三级可展开的树形表格展示所有柜组及其绑定的仪表和通道：

- **第一级：** 柜组信息（名称、区域、初始重量、当前重量、状态、报警阈值）
- **第二级：** 绑定的仪表信息
- **第三级：** 绑定的通道信息（通道标签、系数、偏移量）

### 5.2 添加柜组

1. 点击"添加柜组"按钮
2. 填写以下信息：
   - **柜组名称** — 柜组的标识名称（如"内圈1柜组"）
   - **所属区域** — 柜组所在区域（如"A区"）
   - **初始重量** — 柜组的基准初始重量（kg）
   - **报警阈值** — 瞬间重量变化超过此值触发报警（kg）
   - **颜色** — 柜组在监视页面中的显示颜色
   - **备注** — 可选说明信息
3. 点击"确定"保存

### 5.3 绑定通道

绑定通道是将称重仪表的物理通道与逻辑柜组关联的关键操作，决定了柜组的实时重量如何计算。

1. 在柜组列表中，点击目标柜组行右侧的"绑定通道"按钮
2. 系统弹出树形多选面板，以"仪表 → 通道"的层级结构展示所有可用通道
3. 勾选需要绑定的通道（可多选），为每个通道设置：
   - **权重系数（coefficient）** — 该通道在重量计算中的权重因子（默认1.0）
   - **偏移量（offset）** — 该通道的偏移补偿值（默认0）
4. 点击"确定"完成绑定

**绑定校验机制：** 系统会自动执行三重校验：
- 检查通道是否已启用
- 提示仪表是否在线
- 检测通道是否已被其他柜组绑定（防止重复绑定）

**重量计算公式：** 柜组当前重量 = Σ(通道值_i × 系数_i) + Σ(偏移量_i)

### 5.4 编辑与删除

- **编辑柜组：** 点击柜组行右侧的编辑图标，修改柜组信息后保存
- **解绑通道：** 在展开的通道列表中，点击通道行右侧的解绑按钮
- **删除柜组：** 勾选要删除的柜组，点击"删除"按钮。删除柜组将同时解除所有通道绑定

## 6. 布局编辑操作

> **注意：** 布局编辑仅限管理员操作。

### 6.1 进入布局编辑页面

点击"布局编辑"菜单项进入布局编辑页面。该页面用于编辑保管库的2D平面图布局，将柜组放置到对应的物理位置上。

### 6.2 导入DXF图纸

1. 点击"导入DXF"按钮
2. 选择AutoCAD DXF格式的保管库平面图文件
3. 系统将自动解析DXF文件并渲染为2D SVG背景图
4. 导入成功后，背景图将显示在编辑区域中

### 6.3 放置柜组

1. 从右侧柜组列表中，将需要放置的柜组拖拽到编辑区域的对应位置
2. 拖拽放置后，可通过拖动调整柜组的精确位置
3. 可调整柜组色块的大小以匹配实际物理尺寸
4. 所有修改实时保存

### 6.4 编辑布局

- **移动柜组：** 直接拖拽柜组色块到新位置
- **调整大小：** 拖拽柜组色块的边缘调整尺寸
- **移除柜组：** 右键点击柜组色块，选择"从布局中移除"
- **缩放视图：** 使用鼠标滚轮或缩放控件调整视图大小
- **平移视图：** 按住鼠标中键或空格键拖拽平移视图

## 7. 报警管理操作

### 7.1 查看报警记录

点击"报警管理"菜单项进入报警管理页面。页面以表格形式展示所有报警记录，包括：

| 字段 | 说明 |
|------|------|
| 报警时间 | 报警首次触发的时间 |
| 柜组名称 | 触发报警的柜组 |
| 报警类型 | 超重报警等 |
| 重量变化 | 触发报警时的重量变化值 |
| 发生次数 | 相同报警的累计触发次数（去重后） |
| 处理状态 | 未处理 / 已确认 / 已忽略 / 自动解除 |

### 7.2 筛选报警

页面顶部提供以下筛选条件：

- **状态筛选：** 全部 / 未处理 / 已确认 / 已忽略 / 自动解除
- **柜组筛选：** 按柜组名称筛选
- **时间范围：** 按起止日期筛选
- **关键词搜索：** 搜索报警描述中的关键词

### 7.3 处理报警

1. 在报警列表中找到需要处理的报警记录（状态为"未处理"）
2. 点击报警行右侧的操作按钮：
   - **确认：** 表示已知悉并处理该报警，需填写处理备注说明采取的措施
   - **忽略：** 表示该报警为误报或无需处理，需填写忽略原因
3. 处理完成后，报警状态将更新，处理记录将写入报警处理日志

### 7.4 报警自动解除

当柜组重量恢复到正常范围内时，系统会自动将对应的报警状态更新为"自动解除"。自动解除的报警仍会保留在记录中，供后续审查。

## 8. 数据记录查看

### 8.1 进入数据记录页面

点击"数据记录"菜单项进入数据记录页面。该页面展示所有柜组的历史重量变化记录。

### 8.2 记录内容

每条记录包含以下信息：

| 字段 | 说明 |
|------|------|
| 记录时间 | 重量变化发生的时间 |
| 柜组名称 | 发生变化的柜组 |
| 变化前重量 | 变化前的重量值（kg） |
| 变化后重量 | 变化后的重量值（kg） |
| 变化量 | 重量变化的绝对值（kg） |
| 是否报警 | 该次变化是否触发了报警 |

### 8.3 筛选与搜索

- **按柜组筛选：** 在柜组下拉框中选择特定柜组，仅查看该柜组的记录
- **关键词搜索：** 在搜索框中输入关键词进行模糊搜索
- **分页浏览：** 使用页面底部的分页控件浏览大量记录

### 8.4 数据写入规则

系统自动记录重量变化，写入规则如下：
- 重量变化量超过 0.001 kg 才会写入记录
- 同一柜组两次记录之间至少间隔 5 秒，避免频繁写入
- 记录自动标记是否触发了报警

## 9. 数据分析操作

### 9.1 进入数据分析页面

点击"数据分析"菜单项进入数据分析页面。该页面提供多维度的数据可视化分析功能。

### 9.2 分析维度

数据分析页面提供以下分析维度：

- **历史重量趋势图：** 以折线图展示选定柜组在指定时间范围内的重量变化趋势
- **重量变化频次统计：** 以柱状图展示各柜组的重量变化次数对比
- **设备状态占比：** 以饼图展示设备在线/离线的比例分布
- **报警类型与时间分布：** 以图表展示报警事件的类型分布和时间分布规律

### 9.3 操作方式

1. 在页面顶部选择分析维度和时间范围
2. 选择需要分析的柜组（可多选）
3. 系统自动生成对应的可视化图表
4. 可通过鼠标悬浮查看图表中的具体数据点

## 10. 审计日志查看

> **注意：** 审计日志仅限管理员查看。

### 10.1 进入审计日志页面

点击"审计日志"菜单项进入审计日志页面。该页面记录系统中所有关键操作的完整审计轨迹。

### 10.2 日志内容

每条审计日志包含以下信息：

| 字段 | 说明 |
|------|------|
| 操作时间 | 操作发生的精确时间 |
| 操作人 | 执行操作的用户名称 |
| 操作类型 | 创建 / 更新 / 删除 / 登录 / 其他 |
| 操作对象类型 | 网关 / 仪表 / 柜组 / 用户 等 |
| 操作对象ID | 被操作对象的唯一标识 |
| 变更摘要 | 操作的简要描述 |

### 10.3 筛选与搜索

- **按操作类型筛选：** 选择特定的操作类型进行筛选
- **按操作人筛选：** 选择特定用户查看其操作记录
- **时间范围筛选：** 按起止日期限定查询范围

## 12. 系统设置操作

> **注意：** 系统设置仅限管理员操作。

### 12.1 进入系统设置页面

点击"系统设置"菜单项进入系统设置页面。该页面用于配置数据自动清理策略和查看数据库统计信息。

### 12.2 查看数据统计

页面顶部展示各数据表的当前记录数统计：

| 数据表 | 说明 | 默认保留策略 |
|--------|------|-------------|
| 采集数据（collectionData） | 传感器原始采集数据 | 30天 / 500万条 |
| 重量变化记录（weightChangeRecords） | 柜组重量变化历史 | 365天 / 200万条 |
| 报警记录（alarmRecords） | 报警事件记录 | 365天 |
| 审计日志（auditLogs） | 操作审计记录 | 365天 |

### 12.3 配置自动清理

1. **启用自动清理：** 将"自动清理"开关设为开启状态
2. **设置执行时间：** 选择每天自动执行清理的时间（建议设在业务低峰期，如凌晨2:00~4:00）
3. **配置保留策略：** 为每张数据表分别设置：
   - **保留天数** — 超过此天数的记录将被自动清理
   - **最大记录数** — 记录数超过此上限时，最早的记录将被清理
4. 点击"保存设置"使配置生效

### 12.4 手动清理

如需立即清理数据（例如首次部署后清理积压数据），可点击"立即清理"按钮手动触发一次清理操作。清理过程中页面会显示进度提示。

## 13. 常见问题与故障排除

### 13.1 连接相关问题

**问题：实时监视页面显示"离线"状态**

可能原因及解决方法：
1. **WebSocket连接断开** — 检查页面顶部的连接状态指示器。如果显示红色，尝试刷新页面（F5或Ctrl+Shift+R）
2. **采集服务未运行** — 联系管理员检查采集服务状态。在服务器上执行 `sudo systemctl status weighing-collection.service` 查看服务状态
3. **网关网络不通** — 检查RS485网关的网络连接，确认IP地址和端口配置正确

**问题：柜组显示灰色（离线）**

可能原因：
1. 柜组未绑定任何通道
2. 绑定的仪表通道已禁用
3. 仪表所在的COM端口网络不通
4. 仪表从站地址配置错误

### 13.2 数据相关问题

**问题：柜组当前重量显示为0或异常值**

排查步骤：
1. 进入"仪表管理"页面，检查对应通道的当前值是否正常
2. 进入"柜组管理"页面，检查通道绑定关系是否正确
3. 检查绑定通道的系数和偏移量设置是否合理
4. 确认传感器硬件连接是否正常

**问题：数据记录页面为空**

可能原因：
1. 柜组未绑定通道，没有实时数据产生
2. 重量变化量未超过0.001kg的最小记录阈值
3. 数据已被自动清理（检查系统设置中的保留策略）

### 13.3 报警相关问题

**问题：应该报警但没有触发**

排查步骤：
1. 检查柜组的报警阈值设置是否合理（过大的阈值可能导致报警不触发）
2. 确认柜组绑定的通道是否正常采集数据
3. 检查报警去重机制——如果同一柜组已有未处理的活跃报警，新的相同类型报警将被去重

**问题：报警声音不响**

排查步骤：
1. 检查实时监视页面的声音开关是否已开启
2. 检查浏览器是否允许该网站播放音频（浏览器可能默认阻止自动播放）
3. 检查电脑音量设置

### 13.4 权限相关问题

**问题：看不到某些菜单项**

这是正常现象。操作员角色仅能看到有权限的菜单项。如需访问更多功能，请联系管理员提升权限。

**问题：操作时提示"权限不足"**

当前登录用户的角色没有执行该操作的权限。请联系管理员确认权限分配。
