# 00 — 项目概述

## 产品目标

称重系统管理平台是一套面向金融保管库（如银行金库）的**实时称重监控与管理系统**。其核心目标是通过 RS485 网关连接多台称重仪表，实时采集保险柜组的重量数据，在 Web 界面上以 DXF 平面图的形式可视化展示各柜组的称重状态，并在重量异常变化时触发报警。

## 核心功能

| 功能模块 | 说明 |
|----------|------|
| **实时监视** | 基于 DXF 平面图的保管库可视化，柜组按颜色区分，悬停显示重量详情 |
| **网关管理** | 管理 RS485 串口服务器（如 ZLAN6808），配置 IP、端口、COM 口参数 |
| **仪表管理** | 管理称重仪表（DY7001/DY7004），配置从站地址、通道参数、校准系数 |
| **柜组管理** | 定义保险柜组，绑定仪表通道，配置报警阈值 |
| **布局编辑器** | 导入 DXF 平面图，选择柜列绑定柜组，支持整图旋转，保存并激活布局 |
| **数据记录** | 记录重量变化历史，支持按时间范围查询 |
| **报警管理** | 阈值超限报警、设备离线报警，支持确认处理 |
| **数据分析** | 重量趋势图表、统计分析 |
| **用户管理** | 管理员/操作员角色，模块级权限控制 |
| **审计日志** | 记录所有关键操作（增删改、绑定变更等） |

## 系统边界

### 已实现

本平台已完整实现 **Web 管理界面**和**后端 API 层**，涵盖设备配置、柜组管理、布局编辑、数据查询、权限控制等全部管理功能。数据库 Schema 已设计完成并包含实时数据字段（`currentValue`、`currentWeight`、`status` 等）。

### 尚未实现（需后续开发）

**RS485 数据采集服务**是当前系统的主要缺失部分。该服务负责通过网关连接仪表、按 Modbus RTU 协议轮询读取称重数据、写入数据库。具体包括：

- Modbus RTU 主站通信程序（连接网关 TCP 端口，发送/接收 Modbus 帧）
- 轮询调度器（按配置的 COM 端口参数和仪表地址循环采集）
- 数据写入管道（将采集到的原始值经校准计算后写入 `instrumentChannels.currentValue`）
- 柜组重量聚合（根据绑定关系计算柜组总重量，更新 `cabinetGroups.currentWeight`）
- 报警判定逻辑（比较重量变化与阈值，生成报警记录）
- WebSocket 推送（将实时数据推送到前端，避免轮询）

### 运行形态

当前系统以**单进程 Node.js 应用**运行，前后端合并部署。生产环境建议在前端增加 Nginx 反向代理。未来采集服务可作为独立进程运行，通过数据库或消息队列与 Web 服务通信。
